# Recommender model artifacts

Generated by `export_recommender_model.py` (repository root). These artifacts let
the browser reproduce the source-side NT-book reranker from
`golden_path_quality_prediction_paper.tex` /
`nt_book_drafting_order_gbm.ipynb` **with no backend**. Do not hand-edit;
re-run the script to regenerate.

## What the model is

A gradient-boosting reranker (scikit-learn `GradientBoostingRegressor`,
n=200, depth=2, lr=0.05, subsample=0.8, **8 seeds averaged**) that predicts
*within-group standardized* case-normalized chrF++ draft quality from
source-side features only. It is a **relative, within-pool ranking** model, not
an absolute-quality forecaster.

Trained on **4,220 rows** = `all_scores.csv` (single training
passage, the paper's 840 rows) + `all_scores_2.csv` (multi-passage "A + B + C"
training side, 3,380 rows). The multi-passage rows teach the model the
**union-of-several-books** regime the UI uses when a team selects more than one
already-translated book, rather than extrapolating into it from single-passage
data.

Validation (reproduced at export time): leave-one-direction-out
Spearman = **0.769** overall (single-passage groups
0.770, multi-passage groups 0.769); the paper Table
reports ~0.775 for the single-passage sim+intrinsic GBM.

## Feature set (21 features, in `gbm.json.featureOrder`)

- Similarity (verse TF-IDF, word + char(3,5)): w_top10, w_q95, w_mean, w_tgtbest, w_cov, c_top10, c_tgtbest
- Coverage: cov_freq, cov_type, cov_count_weighted
- Intrinsic candidate difficulty: te_ttr, te_vlen, te_ntok, te_selfred, te_entropy, te_cross_entropy, te_kl, te_pn_density, te_pn_unique_density, te_pn_density_no_top10, te_pn_unique_no_top10

`same_testament` and the `te_cap` capitalization proxy are intentionally
excluded (so candidates and selected books may cross testaments / superseded by
the real proper-noun inventory).

## Files

- `gbm.json` — 8 models x 200 depth-2 trees as flat node arrays, plus per-model
  `init` constant and `learning_rate`; `featureOrder`; chrF++ `calibration`.
  Prediction = mean over models of `init + lr * sum_trees(value at reached leaf)`.
- `books.json` — all 66 Bible books in canonical order, with display
  name, testament, section,
  verse count, and the 11 intrinsic features (constant per candidate book).
- `tgtmax_word.u8`, `tgtmax_char.u8` — per ordered pair (s != c, 4290 pairs),
  the per-candidate-verse best similarity to book s's verses, quantized to
  uint8 (value/255). Concatenated; pair p starts at `tgtmaxOffsets[p]`, length
  `verseCounts[c]`.
- `hist_word.u32`, `hist_char.u32` — per pair, a 128-bin histogram
  (uint32 counts) of the full train x test similarity window over [0,1].
  Pair p occupies `[p*128, (p+1)*128)`.
- `pairs_index.json` — `pairIndexMatrix[s][c]` -> pair index (-1 on diagonal),
  `tgtmaxOffsets`, per-pair `wSum` (window sum, for `w_mean`) and `wCovCount`
  (# train verses with max sim >= 0.4, for `w_cov`), plus
  `verseCounts`, `bins`, `covThreshold`, `tgtmaxScale`.
- `tokens.json` — interned token table + per-book `[tokenId, count]` lists, used
  for the coverage features (`cov_freq`, `cov_type`, `cov_count_weighted`).
- `parity_fixtures.json` — ground-truth `predicted_z` per candidate for
  18 selected-book sets, produced by the notebook's exact
  `rank_candidates`. The TS port asserts equality against these.

Total artifact size: ~10.48 MB raw (compresses well over HTTP).

## Union algebra (how arbitrary multi-book selection works client-side)

`rank_candidates` uses the **union** of selected books as the training side.
For a selected set S and candidate c, each deployment feature recombines from
the per-(s, c) primitives:

- `w_tgtbest` = mean over c's verses of max over s in S of `tgtmax_word[s][c]`
  (elementwise max across selected books). `c_tgtbest` likewise.
- `w_cov` = (sum over s of `wCovCount[s][c]`) / (sum over s of `verseCounts[s]`).
- `w_mean` = (sum over s of `wSum[s][c]`) / (verseCounts[c] * sum over s of `verseCounts[s]`).
- `w_top10`, `w_q95`, `c_top10` = top-10%-mean / 95th-pct of the **summed**
  histograms over S (the only approximated features; histogram error measured at
  export and absorbed by the parity tolerance).
- `cov_*` = computed from the union of selected books' token-count maps.
- intrinsic `te_*` = constant per candidate, read straight from `books.json`.

Decomposition verified exact at export (max abs error
7.6e-17 on exactly-decomposed features); histogram-approx features
max abs error 1.7e-03; resulting `predicted_z` max abs error
2.1e-01.

## Important framing (from the paper)

- Output is a **relative ranking / triage shortlist**, not absolute quality, and
  predicts *reference-overlap* (chrF++) quality specifically.
- The model was validated for ranking candidates from **one** starting passage.
  Selecting many books grows the training side beyond the model's support and
  overlap features saturate; the **first few** recommendations are the reliable
  signal. Treat as a decision aid, not a replacement for translator judgment.
